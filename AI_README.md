# 🤖 AI & Agent Workspace Context`n`nThis repository has been structured to maintain high readability and ease of access for both human developers and AI assistants.`n`n## 📁 Directory Structure Overview`n`n- **`_ai_context/`**: The central brain and data hub for AI operations.`n  - `data/`: Contains JSON datasets, scraped information, Python syntax extractions, and meta-data lists.`n  - `reports/`: Contains analysis results, Markdown comparison reports (e.g., LEET validation), and generated text logs.`n  - `scripts/`: Reserved for utility scripts specifically used for data manipulation by AI.`n- **`notion_automation/`**: Contains Python scripts dedicated to interacting with the Notion API (Read/Update/Create pages, chunking logic).`n- **`gitp/`**: User's scratchpad and algorithmic problem-solving codes (Python).`n- **`01. Areas/`**, **`02. Resources/`**, **`03. Notion/`**: Markdown-based knowledge base and structured notes.`n- **`04. Tools/`**: General utility tools and helper scripts.`n`n## 📝 Rules of Engagement for AI`n1. **Data Operations**: Any intermediate JSON/TXT files generated during analysis MUST be saved in `_ai_context/data/` or `_ai_context/reports/`. Do not clutter the root directory.`n2. **Scripting**: Put Notion-related API code in `notion_automation/`. Put general data-crunching or AI helper scripts in `_ai_context/scripts/`.`n3. **Full Permissions Mode**: AI agents are granted full permission to refactor, write scripts, and execute logic as requested, keeping this folder structure clean.`n
