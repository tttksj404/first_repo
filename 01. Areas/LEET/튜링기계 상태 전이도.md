# 🛰️ [Machine Logic] 튜링기계 상태 전이도 및 알고리즘 연계

## 1. 튜링기계(Turing Machine)의 추리논증적 의미
- **정의**: 유한한 상태(State)와 테이프(Memory)를 가지고, 규칙(Transition Function)에 따라 상태를 변화시키는 추상적 모델.
- **LEET 출제 포인트**: 복잡한 기계표를 읽고, 입력값에 따라 상태가 어떻게 변하는지 'Trace(추적)'하는 능력을 측정. (31번 킬러 문항 빈출)

---

## 2. SSAFY 알고리즘 연동: 상태 배열 관리
- **핵심 기술**: [[SSAFY 알고리즘 - 상태 배열 관리]]
    - **격자 제어 (Grid Control)**: 2차원 배열 상에서의 이동과 튜링기계 테이프의 이동은 논리적으로 동일함.
    - **상태 전이 (State Transition)**: `next_state = transition_table[current_state][input]`
- **실전 팁**:
    - 복잡한 전이도는 표로 그려서 한 눈에 파악한다.
    - SSAFY의 BFS/DFS 탐색 시 '방문 체크(Visited)' 배열이 튜링기계의 '상태 기록'과 같음을 인지한다.

## 3. 고득점 전략 (140+ Target)
1. **Machine Logic Simulation**: 머릿속에서 돌리지 말고, 여백에 '상태 변화'를 기록한다. (Trace Table 작성)
2. **종료 조건**: 기계가 멈추는(Halting) 조건을 선지 분석 전에 미리 확정한다.
