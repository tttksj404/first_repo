# 🦈 실전 상세 분석: 상어 시리즈 (낚시왕 & 마법사 상어)

삼성 A형의 단골 손님인 '상어 시리즈'는 **"수많은 객체가 각자의 속도로 이동하고, 겹쳤을 때 잡아먹는"** 시뮬레이션의 정점입니다.

---

## 1. [데이터 설계] 왜 딕셔너리(`dict`)인가?
격자가 100x100이고 상어가 10마리라면, 격자 전체를 도는 것보다 상어 10마리만 순회하는 것이 훨씬 빠릅니다.

```python
# shark_id: [r, c, speed, direction, size]
sharks = {
    1: [2, 3, 4, 1, 10], # 1번 상어의 정보
    2: [0, 1, 2, 3, 5]   # 2번 상어의 정보
}
```
이렇게 관리하면 `sharks.keys()`만 돌면 되므로 시간 복잡도 관리가 매우 쉬워집니다.

---

## 2. [이동의 기술] 속도 최적화 (The Modulo Trick)
상어의 속도가 1000이고 격자가 5x5일 때, 1000번 칸을 이동시키면 시간 초과로 탈락합니다. 상어는 결국 제자리로 돌아오기 때문입니다.

```python
# 상하(R) 또는 좌우(C) 이동 시 주기 계산
# N-1 칸을 갔다가 다시 N-1 칸을 돌아오면 제자리 (총 2 * (N-1) 칸)
def get_next_pos(r, c, s, d, R, C):
    if d <= 1: # 상하 이동
        cycle = 2 * (R - 1)
        s %= cycle # 불필요한 바퀴 수 제거
        for _ in range(s):
            nr = r + dr[d]
            if not (0 <= nr < R):
                d = 1 - d # 방향 반전
                r += dr[d]
            else:
                r = nr
    else: # 좌우 이동
        cycle = 2 * (C - 1)
        s %= cycle
        ...
    return r, c, d
```
> [!info] **나머지 연산의 마법**
> 이 `s %= cycle` 한 줄이 삼성 시험에서 파이썬 응시자를 합격시키는 '신의 한 수'입니다.

---

## 3. [상태 관리] 동시 이동과 충돌 처리
상어들이 동시에 움직여서 한 칸에 모였을 때, 가장 큰 놈만 남겨야 합니다.

```python
# 1. 새로운 격자(new_board)에 이동한 상어들의 ID를 담습니다.
new_board = [[[] for _ in range(C)] for _ in range(R)]

for s_id in list(sharks.keys()):
    r, c, s, d, z = sharks[s_id]
    nr, nc, nd = get_next_pos(r, c, s, d, R, C)
    
    # 해당 칸에 상어 ID 추가
    new_board[nr][nc].append(s_id)
    # 상어 정보 갱신
    sharks[s_id] = [nr, nc, s, nd, z]

# 2. 한 칸에 두 마리 이상인 경우 '가장 큰 놈'만 남기기
for r in range(R):
    for c in range(C):
        if len(new_board[r][c]) > 1:
            # 크기(z) 기준 내림차순 정렬
            new_board[r][c].sort(key=lambda x: sharks[x][4], reverse=True)
            
            # 1등 상어를 제외한 나머지(losers) 삭제
            winner = new_board[r][c][0]
            losers = new_board[r][c][1:]
            for l_id in losers:
                del sharks[l_id] # 딕셔너리에서 즉시 제거
```

## 4. [A형 실전 팁] 격자 크기와 인덱스
- **1번 인덱스 주의**: 삼성 문제는 좌표를 `(1, 1)`부터 주는 경우가 많습니다. 입력을 받을 때 `r-1, c-1`을 하거나, 격자 크기를 `(R+1, C+1)`로 만들어 헷갈림을 방지하세요.
- **방향 정의**: 문제마다 `1:상, 2:하, 3:우, 4:좌` 등으로 번호가 다릅니다. 이 번호를 그대로 `dr, dc` 리스트의 인덱스와 매칭하는 연습을 하세요.
