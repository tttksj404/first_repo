# 🎯 삼성 A형 취득을 위한 백지 코딩 정복 가이드 (IM → A형 도약편)

이 가이드는 IM 수준의 학생이 **"아무런 힌트 없이 백지에서 삼성 A형 정답 코드를 100% 스스로 완성"**할 수 있도록 사고의 흐름을 5배 이상의 분량으로 상세히 기록한 것입니다.

---

## 1. [철학] IM과 A형의 결정적 차이 (왜 IM 실력으로는 A형이 안 될까?)
IM 등급은 **"단순 구현"**입니다. 격자를 돌리고, 이동시키면 끝납니다. 하지만 A형은 **"복합 작용"**입니다. 
- 상어가 이동하면서 **동시에** 잡아먹고, 
- 파이어볼이 흩어지면서 **동시에** 합쳐지고, 
- 핀볼이 벽에 부딪히면서 **동시에** 방향이 꺾입니다. 

**A형 합격의 열쇠는 "동시성(Concurrency)"과 "상태 관리(State Management)"**입니다. 이 가이드는 이 두 가지를 백지에서 어떻게 코드로 녹여내는지 아주 상세히 다룹니다.

---

## 2. [설계의 정석] 문제를 읽고 10분 동안 키보드를 치지 마세요.
A형은 코딩이 아니라 **'설계'**에서 당락이 결정됩니다. 문제를 읽으며 다음 3가지를 반드시 종이에 적으세요.

### 2-1. 데이터 구조 (Data Structure): "어디에 담을 것인가?"
- **2차원 배열(`board`)**: 숫자가 하나만 있는 격자 (벽돌 깨기, 미세먼지)
- **딕셔너리(`dict`)**: 움직이는 객체가 많고, 좌표 관리가 중요한 경우 (낚시왕 상어)
- **3차원 리스트(`board[r][c] = []`)**: 한 칸에 여러 객체가 중첩될 수 있는 경우 (마법사 상어와 파이어볼)

### 2-2. 시뮬레이션 순서 (Flow): "누가 먼저 움직이는가?"
문제에 적힌 번호를 그대로 번호 매겨보세요. 
1. 이동한다 → 2. 벽에 부딪힌다 → 3. 잡아먹는다. 
**이 순서가 1번만 바뀌어도 답은 0점이 됩니다.**

---

## 3. [심화 분석] 핀볼 게임(5650) 백지 정복 (사고의 징검다리)

핀볼 게임은 A형의 **'방향 전환'**과 **'시뮬레이션'**의 정수입니다. 이 문제를 백지에서 짜려면 다음 단계가 필요합니다.

### 3-1. 방향 테이블의 설계 (Direction Mapping)
상하좌우를 `0, 1, 2, 3`으로 정하세요. 그리고 1~5번 블록마다 들어오는 방향에 따라 나가는 방향이 어떻게 바뀌는지 **표로 만드세요.**
- **1번 블록**: `[1, 3, 0, 2]` 
    - 상(0)으로 들어오면 하(1)로 반사.
    - 하(1)로 들어오면 우(3)로 꺾임.
    - 좌(2)로 들어오면 상(0)으로 꺾임.
    - 우(3)로 들어오면 좌(2)로 반사.
- **이걸 `if/elif` 100줄로 적는 학생은 무조건 틀립니다.** 반드시 리스트 형태의 테이블로 만드세요.

### 3-2. 웜홀(Wormhole)의 처리
웜홀은 항상 **쌍(Pair)**으로 존재합니다. 
- 웜홀의 좌표를 입력을 받을 때 미리 `wormholes = { 6: [(r1, c1), (r2, c2)], ... }` 형태로 담으세요. 
- (r, c) 좌표가 (r1, c1)이라면 (r2, c2)로 즉시 이동시키는 코드를 짜야 합니다. 

### 3-3. 무한 루프의 탈출 조건
- 구슬은 **"시작점으로 돌아오거나"** 또는 **"블랙홀(-1)을 만날 때까지"** 멈추지 않습니다.
- `while True:` 안에서 이 두 조건을 가장 먼저 체크해야 합니다.

## 4. [디버깅] "예제는 맞는데 제출하면 80%만 맞는 이유"
삼성 시험의 가장 큰 함정입니다. 80%에서 막혔을 때 다음을 체크하세요.

### 4-1. 경계값(Boundary) 처리
- 격자 끝에 닿았을 때 방향을 바꾸고 **"그 자리에서 한 번 더 연산"**을 해야 하나요? 
- 아니면 **"방향만 바꾸고 다음 턴에 이동"**해야 하나요? 문제의 단 한 단어를 놓치면 80%에서 멈춥니다.

### 4-2. 원본 보존의 법칙 (Deep Copy)
- "모든 상어가 동시에 이동한다"는 조건이 있다면, `board`를 직접 수정하지 마세요. 
- 새로운 `new_board`를 만들어 결과를 다 적은 뒤, 마지막에 `board = new_board`로 덮어씌워야 합니다.

## 5. [실전 예시] 낚시왕 상어 (Fisher & Shark) - 분량 2배 확장 분석
이 문제는 **"이동 + 잡아먹기 + 속도 최적화"**의 끝판왕입니다.

### 5-1. 속도 최적화 (The Speed Trick)
- 상어의 속도가 1000이고 격자가 5x5라면? 1000번 `while`문을 돌리면 시간 초과입니다.
- **나머지 연산(%) 활용**: `s = s % (2 * (N-1))` 수식을 사용하세요. 상어가 한 바퀴 돌아 제자리로 오는 주기를 계산하여 불필요한 연산을 지우는 것이 **A형 취득의 필수 조건**입니다.

### 5-2. 상어 잡아먹기 (The Winner Takes All)
- 이동 후 한 칸에 상어가 여러 마리라면? 
- `new_board[r][c].append(shark_id)`로 일단 다 담은 뒤, 루프를 돌며 가장 큰 상어만 남기고 나머지는 `sharks` 딕셔너리에서 지워버리세요.
